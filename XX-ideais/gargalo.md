O gargalo da engenharia de software não é mais escrever código.

O humano é necessário para gerar ideias do que precisa ser feito (o que) e determinar os limites (como).

O humano precisa ter capacidade técnica para dizer para a máquina como ela deve validar a construção do software.

Traçando paralelos, a engenharia de software agora tem uma relação com os agentes (LLMs/chatgpt/claude code/etc) algo similar ao que a engenharia civil tem com os pedreiros. Uma diferença importante é que a engenharia civil é regulada e a engenharia de software não. Um software não precisa de um responsável técnico para ser construído, uma casa sim. Um engenheiro de software não precisa apresentar o CREA para construir um aplicativo, uma ponte sim. O engenheiro de software não precisa seguir a NR-10 para criar um e-commerce, um galpão sim.
